A Caution Against Using HTML to Style

Semantic Inline Elements

Although some elements may appear to have some default styling by the browser, HTML should always be used semantically, meaning that they are used based on the purpose they serve instead of their aesthetic look. It's imperative for you to use CSS for all aesthetic purposes.

This is some text that you should look at, but isn't necessarily important. You also need to understand that <em> is used to emphases words or a phrase that are more important than their surroundings.

Some elements are useful when styling

When a piece of content has no specific purpose, or there is no other better semantic HTML element to use, <div> and <span> can be quite useful.
